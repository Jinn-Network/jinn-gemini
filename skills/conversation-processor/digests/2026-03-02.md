# Contributors Update — Mar 2, 2026

> Weekend RPC outage exposed infra gaps. This week: stabilize, unblock operators, and start shaping a consumer content product.

## What happened

- **RPC quota exhaustion**: Tenderly quota hit over the weekend, taking down the indexer and breaking content delivery on the frontend. Quota reset with the new month. We've deployed `rpc.jinn.network` as a centralized RPC proxy with fallback support to prevent this from recurring.
- **Ponder rebuilt**: Production Ponder got stuck and had to be recreated. Currently backfilled from only 2 days ago -- needs extending to ~2 weeks so artifacts actually show up on [app.jinn.network](https://app.jinn.network). Custom domain made the swap invisible to clients.
- **Content streams not delivering**: Some ventures start but aren't delivering artifacts. Unclear whether it's the template or network instability from the above issues. Under investigation.

## What shipped last cycle

- **Multi-contract staking**: Worker filtering supports both V1 and V2 staking contracts. Heartbeat is contract-dependent -- no heartbeat on V2 (delivery-based). (Ritsu)
- **Auto-restaking** (JINN-393): Done -- evicted services auto-restake. JINN-392 marked duplicate.
- **Content feed convention** (JINN-408): `feed:{topic}` type markers on output artifacts. On branch, moved to In Review.
- **RPC proxy**: `rpc.jinn.network` deployed, all consumers migrated. Ponder multi-RPC fallback added.

## Architecture decisions

- **Orchestrator scope narrowed**: Agreed to focus on proving invariant enforcement with simple, concrete goals before pursuing autonomy. We don't yet have non-autonomous ventures doing reliably useful work -- that comes first. Research continues but is not part of go-to-market.
- **Personal content feed as product direction**: A consumer-facing product where individuals get personalized content feeds. Consumer side should feel like web2 (no downloads, no crypto). Could be a two-sided marketplace: template creators + consumers. Not building this week, but shaping the approach.
- **Railway consolidation**: Proposal to move workers into the same Railway project for egress savings and PR environment support. Would help with the current deployment pain (slow builds, no visibility).

## Open questions

- **Content feed product UX**: Consumers won't download an app for a news feed. What's the right lightweight web experience? How does monetization work -- x402 for premium feeds? Free tier?
- **Ponder per node**: Should each node eventually run its own Ponder? Adds redundancy but creates networking/discovery problems. Could leverage private IPFS P2P layer.
- **V1 deprecation timeline**: When do we stop supporting the original staking contract? No rush -- workers handle both -- but need to decide when to push everyone to V2.
