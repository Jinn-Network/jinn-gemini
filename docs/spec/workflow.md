# Current Development Workflow and Processes

Our current process looks like this:

* Daily meetings and occasional in-person meetings where we discuss the previous periods' work and what we are planning to move forward to.
* Recently we have moved to a set up where by we manually break down the work and create linear tickets. Tickets in the Linear app.
* From there, we pull down those Linear issues and use some combination of coding agents to execute those tasks. If they're bigger tasks, then we will create a specification document.
* Historically, we've done that locally, but more recently, we've been experimenting with doing that in Linear.
* We will usually create one branch for each of our work, and we will push to that branch before merging it down to main, and we will roughly keep Linear in sync through that process.
* We have a set of personal prompts and commands that we use, particularly within Cursor, to create the specification documents and to manage the implementation of work, but they're not currently shared between team members.
* In terms of other non-coding related tasks, there's very little structure around it currently. In terms of ongoing research, we will effectively discover things here and there which we'll share on Slack and historically on Telegram, but we more recently moved to Slack, and you'll share them as links or just refer to them within calls.
* In terms of operations, we have recently also started using Slack because we feel that it's apps, including Fireflies and Cursor integrations, could be beneficial.

I think that's basically the process so far.
