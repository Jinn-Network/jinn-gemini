# Rule 1: Never Commit Secrets to the Repository

## Context

In an autonomous agent system handling real cryptocurrency and making blockchain transactions, leaked secrets can result in immediate, irreversible financial loss. Once committed to git, secrets remain in the repository history forever—even if deleted in later commits.

**Critical secrets in this codebase:**
- **Agent private keys**: Control on-chain Safe multisigs with OLAS tokens and ETH
- **Master wallet keys**: Create and manage OLAS services
- **API keys**: Gemini API, Supabase service role, RPC providers
- **Passwords**: OLAS middleware (`OPERATE_PASSWORD`)

A single leaked agent key can drain all funds from its Safe. This is a **hard constraint with no exceptions**.

- Git history is permanent—even if you delete the file later

## Key Principles

### 1. **No Secrets in Source Code**
- Never hardcode credentials in `.ts`, `.js`, `.json`, `.yaml`, `.toml` files
- Never include secrets in comments (they're searchable in git history)
- Use environment variables or secure secret management systems

### 2. **Gitignore Sensitive Files**
- `.env` files (except `.env.example`)
- Key files (`*_private_key.txt`, `*.pem`, `*.key`)
- Wallet directories (`olas-operate-middleware/.operate/wallets/`)
- Agent key storage (`olas-operate-middleware/.operate/keys/`)

### 3. **Never Log Secrets**
- Don't log environment variables that contain secrets
- Don't log full error objects (may contain auth headers)
- Don't log transaction payloads that include private keys
- When logging addresses, only log public addresses (not private keys)

### 4. **Sanitize Before Sharing**
- Remove secrets from logs before sharing in issues/PRs
- Use placeholder values in documentation and examples
- Rotate any key that might have been exposed

### 5. **Validation Over Documentation**
- Require environment variables at startup (fail fast if missing)
- Never use default/placeholder secrets in production
- Automated checks (pre-commit hooks, CI) to detect secrets

## Real-World Impact in This Codebase

### Incident Example (Hypothetical)

```typescript
// Agent key accidentally committed in PR #123
const agentKey = '0xabcd...'; // This Safe has 150 OLAS + 0.05 ETH

// Even if deleted in next commit, key is in git history
// Result: Attacker finds key, drains Safe, 150 OLAS lost
```

### Recovery Cost
- **If caught before push**: Zero cost (amend commit, force push to branch)
- **If caught after merge**: Must rotate all keys, redeploy services (~2 hours + gas costs)
- **If exploited**: Permanent loss of funds in compromised Safe (non-recoverable)

## Enforcement

### Pre-commit Hook (Recommended)
```bash
#!/bin/bash
# .git/hooks/pre-commit

# Detect common secret patterns
if git diff --cached | grep -E '(PRIVATE_KEY|API_KEY|SECRET|PASSWORD)\s*=\s*["\x27][^"\x27]{20,}'; then
  echo "❌ Potential secret detected in staged changes"
  echo "Never commit secrets to git. Use environment variables."
  exit 1
fi
```

### Manual Review Checklist
Before committing:
- [ ] No hardcoded keys, tokens, or passwords
- [ ] No secrets in comments or documentation
- [ ] `.env` files in `.gitignore`
- [ ] Only placeholder values in `.env.example`
- [ ] No secrets in log statements

### Emergency: Secret Already Committed

```bash
# If you committed a secret:
git reset --soft HEAD~1  # Undo commit (keep changes staged)
# Remove secret from code
git add .
git commit -m "fix: use environment variable"

# If already pushed to shared branch:
# 1. Rotate the secret immediately (assume compromised)
# 2. Force push to overwrite history (if safe to do so)
# 3. Notify team
```

## Application to AI Code Generation

When AI generates code:
1. **Never provide real secrets to AI**: Use placeholders in prompts
2. **Template outputs**: AI should generate code that reads from env vars
3. **Reject insecure patterns**: Don't accept hardcoded credentials
4. **Validate before commit**: Pre-commit hooks catch AI-generated secrets

The "Never commit secrets" rule prevents AI from learning insecure patterns and propagating them across sessions.
