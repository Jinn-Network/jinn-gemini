# Rule 2: Always Validate On-Chain State Before Financial Operations

## Context

Blockchain transactions are **immutable** and **irreversible**. A transaction submitted to the network cannot be undone—even if it was based on stale data or incorrect assumptions. In this system, failed transactions result in:

- **Wasted gas fees**: Each failed Safe transaction costs ETH
- **Locked funds**: Invalid operations can trap tokens in contracts
- **Debugging complexity**: Failed transactions are harder to troubleshoot than prevented ones
- **User frustration**: Unexpected failures erode trust in the system

**This rule applies to all operations that:**
- Transfer tokens (OLAS, ETH)
- Execute Safe transactions (mech deliveries, staking)
- Deploy or interact with contracts
- Claim rewards or update on-chain state

**Why preflight validation?**
- On-chain state can change between query and transaction submission
- Multiple workers may attempt the same operation concurrently
- Network delays can make local state stale
- Gas costs for failed transactions are non-recoverable

## Test Cases

### Case 1: Mech Request Delivery

#### ❌ Violates Rule

```typescript
// File: worker/mech_worker.ts

// ❌ VIOLATION: No preflight check before delivery
export async function processRequest(request: UnclaimedRequest) {
  const result = await runAgentForRequest(request);

  // Directly submit delivery without checking if already delivered
  await deliverViaSafe({
    requestId: request.id,
    resultContent: result.output,
    targetMechAddress: request.mech,
    safeAddress: SAFE_ADDRESS,
    wait: true
  });
  // If already delivered: wastes gas, transaction reverts
}
```

**Why this is dangerous:**
- Another worker may have delivered while this agent was running
- Network indexer may be delayed (shows undelivered when actually delivered)
- Safe transaction construction + submission costs gas even if it reverts
- Result: Wasted ~0.001 ETH per failed delivery attempt

#### ✅ Follows Rule

```typescript
// File: worker/mech_worker.ts

// ✅ CORRECT: Preflight check before delivery
export async function processRequest(request: UnclaimedRequest) {
  const result = await runAgentForRequest(request);

  // Preflight: Verify request is still undelivered on-chain
  const requestIdHex = String(request.id).startsWith('0x')
    ? String(request.id)
    : '0x' + BigInt(String(request.id)).toString(16);

  const isUndelivered = await isUndeliveredOnChain({
    mechAddress: request.mech,
    requestIdHex,
    rpcHttpUrl: process.env.RPC_URL
  });

  if (!isUndelivered) {
    workerLogger.info(
      { requestId: request.id },
      'Request already delivered, skipping Safe delivery'
    );
    return; // ✅ Avoid wasted gas
  }

  // Safe to proceed with delivery
  await deliverViaSafe({
    requestId: request.id,
    resultContent: result.output,
    targetMechAddress: request.mech,
    safeAddress: SAFE_ADDRESS,
    wait: true
  });
}
```

**Preflight helper implementation:**

```typescript
// ✅ CORRECT: Reusable preflight validation
async function isUndeliveredOnChain(params: {
  mechAddress: string;
  requestIdHex: string;
  rpcHttpUrl?: string;
}): Promise<boolean> {
  const { mechAddress, requestIdHex, rpcHttpUrl } = params;

  try {
    if (!rpcHttpUrl) {
      // If no RPC, best-effort: don't block delivery
      workerLogger.warn('No RPC URL for preflight check');
      return true;
    }

    const web3 = new Web3(rpcHttpUrl);
    const contract = new web3.eth.Contract(
      AgentMechABI,
      mechAddress
    );

    // Query on-chain undelivered set
    const undeliveredIds: string[] = await contract.methods
      .getUndeliveredRequestIds(100, 0)
      .call();

    const undeliveredSet = new Set(
      undeliveredIds.map(id => String(id).toLowerCase())
    );

    return undeliveredSet.has(requestIdHex.toLowerCase());
  } catch (error) {
    // On RPC error, log but don't fail hard
    workerLogger.warn('Preflight check failed', { error });
    return true; // Best-effort: allow delivery attempt
  }
}
```

```typescript
// File: scripts/fund-agent.ts

// ✅ CORRECT: Preflight balance validation
export async function fundAgent(
  agentAddress: string,
  olasAmount: bigint
) {
  const olasToken = new ethers.Contract(
    OLAS_TOKEN_ADDRESS,
    OlasTokenABI,
    signer
  );

  // Preflight: Check sender has sufficient balance
  const senderAddress = await signer.getAddress();
  const balance = await olasToken.balanceOf(senderAddress);

  if (balance < olasAmount) {
    throw new Error(
      `Insufficient OLAS balance: have ${ethers.formatUnits(balance, 18)}, ` +
      `need ${ethers.formatUnits(olasAmount, 18)}`
    );
  }

  // Safe to proceed with transfer
  logger.info('Transferring OLAS', {
    from: senderAddress,
    to: agentAddress,
    amount: ethers.formatUnits(olasAmount, 18)
  });

  const tx = await olasToken.transfer(agentAddress, olasAmount);
  await tx.wait();

  logger.info('Transfer confirmed', { txHash: tx.hash });
}
```

```typescript
// File: worker/OlasStakingManager.ts

// ✅ CORRECT: Preflight service state validation
export async function stakeService(serviceId: number) {
  const stakingContract = new ethers.Contract(
    STAKING_CONTRACT_ADDRESS,
    StakingABI,
    signer
  );

  // Preflight: Check if service is already staked
  const isStaked = await stakingContract.isServiceStaked(serviceId);

  if (isStaked) {
    logger.info({ serviceId }, 'Service already staked, skipping');
    return;
  }

  // Preflight: Check service state on ServiceRegistry
  const serviceRegistry = new ethers.Contract(
    SERVICE_REGISTRY_ADDRESS,
    ServiceRegistryABI,
    signer
  );

  const serviceState = await serviceRegistry.getService(serviceId);
  const isActive = serviceState.state === ServiceState.ACTIVE;

  if (!isActive) {
    throw new Error(
      `Cannot stake service ${serviceId}: state is ${serviceState.state}, ` +
      `must be ACTIVE`
    );
  }

  // Safe to proceed with staking
  logger.info({ serviceId }, 'Staking service');
  const tx = await stakingContract.stake(serviceId);
  const receipt = await tx.wait();

  logger.info({ serviceId, txHash: receipt.hash }, 'Service staked');
}
```

### Case 4: Safe Transaction Construction

#### ❌ Violates Rule

```typescript
// ❌ VIOLATION: No Safe configuration validation
export async function executeSafeTransaction(
  safeAddress: string,
  to: string,
  data: string
) {
  const safeSDK = await Safe.create({
    ethAdapter,
    safeAddress
  });

  // Directly create transaction without validating Safe state
  const tx = await safeSDK.createTransaction({
    safeTransactionData: { to, data, value: '0' }
  });

  const executeTx = await safeSDK.executeTransaction(tx);
  await executeTx.transactionResponse?.wait();
  // If threshold > signers available: transaction fails
}
```

#### ✅ Follows Rule

```typescript
// ✅ CORRECT: Preflight Safe configuration validation
export async function executeSafeTransaction(
  safeAddress: string,
  to: string,
  data: string
) {
  const safeSDK = await Safe.create({
    ethAdapter,
    safeAddress
  });

  // Preflight: Validate Safe configuration
  const threshold = await safeSDK.getThreshold();
  const owners = await safeSDK.getOwners();

  if (threshold > owners.length) {
    throw new Error(
      `Safe ${safeAddress} has invalid configuration: ` +
      `threshold=${threshold} but only ${owners.length} owners`
    );
  }

  // Preflight: Check if signer is an owner
  const signerAddress = await signer.getAddress();
  const isOwner = owners
    .map(addr => addr.toLowerCase())
    .includes(signerAddress.toLowerCase());

  if (!isOwner) {
    throw new Error(
      `Signer ${signerAddress} is not an owner of Safe ${safeAddress}`
    );
  }

  // Safe to proceed with transaction
  logger.info('Creating Safe transaction', {
    safe: safeAddress,
    to,
    threshold,
    ownerCount: owners.length
  });

  const tx = await safeSDK.createTransaction({
    safeTransactionData: { to, data, value: '0' }
  });

  const executeTx = await safeSDK.executeTransaction(tx);
  await executeTx.transactionResponse?.wait();

  logger.info('Safe transaction executed', {
    txHash: executeTx.transactionResponse?.hash
  });
}
```

## Key Principles

### 1. **Query Before Write**
- Always read current on-chain state before submitting a transaction
- Use `view`/`pure` functions (no gas cost) to validate assumptions
- Examples: Check balance, verify state, confirm eligibility

### 2. **Handle Race Conditions**
- Between query and transaction submission, state may change
- Preflight checks reduce but don't eliminate race conditions
- Contracts should use idempotency patterns where possible

### 3. **Fail Fast, Fail Cheap**
- Detect invalid operations **before** constructing transactions
- RPC view calls cost zero gas vs. failed transactions cost real ETH
- Early validation provides better error messages

### 4. **Best-Effort When Unavailable**
- If RPC is unavailable, log warning and allow operation to proceed
- Network issues shouldn't permanently block critical operations
- Let the contract's state validation be the final authority

### 5. **Log All Validations**
- Log both successful and failed preflight checks
- Helps diagnose why operations were skipped or attempted
- Critical for post-incident debugging

## Real-World Impact in This Codebase

### Gas Cost Comparison

| Operation | Preflight Cost | Failed TX Cost | Savings |
|-----------|---------------|----------------|---------|
| Mech delivery | 0 ETH (view call) | ~0.001 ETH | 0.001 ETH |
| Token transfer | 0 ETH (balanceOf) | ~0.0005 ETH | 0.0005 ETH |
| Service staking | 0 ETH (isStaked) | ~0.002 ETH | 0.002 ETH |

**Cumulative savings:**
- 10 prevented failed deliveries = **0.01 ETH saved** (~$30 at current prices)
- 100 prevented operations = **0.1 ETH saved** (~$300)

## Enforcement

### Code Review Checklist

Before submitting code that makes blockchain transactions:
- [ ] Preflight validation implemented for all state assumptions
- [ ] Balance/allowance checked before token transfers
- [ ] Service/contract state verified before state-changing calls
- [ ] Safe configuration validated before Safe transactions
- [ ] Errors logged with context for debugging

## Application to AI Code Generation

When AI generates blockchain interaction code:
1. **Always include preflight checks**: AI should assume state may be stale
2. **Use view functions liberally**: Zero cost, high safety
3. **Handle validation failures gracefully**: Log and skip, don't crash
4. **Learn from failed transactions**: Update patterns to prevent recurrence

The "Always validate" rule prevents AI from generating wasteful transaction-first patterns.
