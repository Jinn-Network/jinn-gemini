# Jinn Agentic Venture Blueprint

The blueprint is structured with a hierarchy of importance – constitution at the top, documentation at the bottom. Each layer should be regularly tested to ensure it does not violate a higher requirement.

## Intent Sections
Set by humans. The *what* and *why* of the venture - the unchanging core that guides all execution decisions.

### Constitution
See [constitution.md](./constitution.md)

### Vision
See [vision.md](./vision.md)

### Requirements
See ./requirements.md

## Execution Sections
Set by agents. The *how* of achieving the intent. This is maintained in the repo by agents and evolves as the system learns and improves. It is broken down into:

- **Specification** – how the system is designed to be implemented (architectural decisions and design patterns)
- **Implementation** – the code and actual implementation (living codebase)
- **Documentation** – a description of the implementation (how the system actually works)
