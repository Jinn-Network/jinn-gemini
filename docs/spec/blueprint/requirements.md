## Observability

The network should maintain three levels of observability:
- human (e.g. frontends)
- programmatic (e.g. scripts)
- agentic (e.g. MCP tools)

At any given moment, it should be possible to observe any requirement in this document via those 3 means.

### Jobs

#### Runs

Each run of a job should consist of:
- efficient search to gather relevant past activity across the Jinn network
- a synthesis of that activity into relevant learnings that will boost the performance of this execution
- the execution of the job
- a successful on-chain delivery, meeting the demands of the initial job request
- sufficient data to assist the network in future activity
