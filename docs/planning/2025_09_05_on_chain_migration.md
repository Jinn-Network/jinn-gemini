# On-Chain Migration: Key Concepts & Resources

This document summarizes the core technologies, protocols, and internal systems relevant to the proposed on-chain migration plan for Project Jinn. It is intended to provide essential context for assessing the plan's technical feasibility.

## 1. Olas Protocol

-   **Core Function:** Olas is a protocol for creating and operating a decentralized economy of autonomous AI services.
-   **Key Components:**
    -   **Registries (ERC-721):** On-chain registries for canonical agents, components, and services. The migration plan correctly maps Jinn's "jobs" to Olas "services."
    -   **Tokenomics (OLAS & veOLAS):** OLAS is the native token. It can be locked for `veOLAS` (vote-escrowed OLAS) to gain governance power.
    -   **Staking & Emissions:** `veOLAS` holders direct OLAS token emissions to different "staking programs" (gauges). This is the core mechanism the plan leverages to incentivize workers (operators) to execute jobs for a specific mission.
-   **"slowlas" Reference:** This refers to the Olas protocol's use of multi-week epochs for calculating and distributing staking rewards, which introduces a delay between staking/voting and the resulting incentive adjustments.
-   **Relevant Documentation:** `docs/documentation/OLAS.md`

## 2. Token Launch & Liquidity: Doppler

-   **Core Function:** Doppler is an EVM-based protocol designed to launch new tokens and bootstrap their liquidity.
-   **Mechanism:** It uses a "Dutch Auction Dynamic Bonding Curve" to manage token creation, price discovery, and the establishment of initial liquidity pools on platforms like Uniswap.
-   **Relevance to Plan:** The migration plan proposes that each new "mission" (staking contract) could also launch its own token to raise capital, addressing the "founder has no capital" problem. Doppler provides a direct, off-the-shelf solution for this complex process.
-   **Relevant Documentation:** [Doppler Docs](https://docs.doppler.lol/)

## 3. Decentralized Storage: Irys

-   **Core Function:** Irys (formerly Bundlr) is a "datachain" optimized for the permanent, affordable, and scalable storage of data on-chain.
-   **Relevance to Plan:** The plan suggests replacing the current database storage of "artefacts" with a decentralized solution. Irys is a strong candidate for this, as it is more cost-effective for large data blobs than storing them directly on an execution-focused blockchain like Ethereum.
-   **Relevant Documentation:** [Irys Docs](https://docs.irys.xyz/)

## 4. Agent Memory Management: Cipher

-   **Core Function:** Cipher is a specialized tool for managing the memory of AI and agentic systems.
-   **Architecture:** It features a "dual memory system":
    -   **System 1 (Explicit Knowledge):** Stores factual information, codebase details, and past interactions using vector search and knowledge graphs.
    -   **System 2 (Reasoning Patterns):** Captures the agent's problem-solving logic to improve its reasoning over time.
-   **Relevance to Plan:** The plan proposes moving agent memories from the Supabase database to a dedicated open-source tool. Cipher is a prime candidate, designed specifically for the needs of complex agentic systems.
-   **Relevant Documentation:** [Cipher Docs](https://docs.byterover.dev/cipher/overview)

## 5. Incentive Abstraction: The Convex Finance Model

-   **Core Function:** Convex Finance provides a layer on top of Curve Finance's `veCRV` (vote-escrowed CRV) system. Users can stake CRV or LP tokens to receive liquid versions of their staked assets and boosted rewards without needing to lock tokens for long periods themselves.
-   **Relevance to Plan:** The plan mentions a "convex style" system as a future idea. This would involve creating a similar abstraction layer over `veOLAS`. It would allow capital providers to easily contribute to Jinn missions and earn rewards without needing to deeply understand the Olas staking and voting mechanics, thus circumventing the "slowlas" lockup periods and complexity.

## 6. Project Jinn: Current Architecture

-   **Core Function:** An autonomous AI agent system for complex, multi-step tasks.
-   **Architecture:**
    -   **Database-Centric:** Supabase (PostgreSQL) is the single source of truth, using an event-driven model orchestrated by database triggers (e.g., `universal_job_dispatcher_v2`). The `events` table acts as a universal event bus.
    -   **Worker/Agent Separation:** Stateless `workers` poll the `job_board` for tasks, which are then executed by an `Agent` that interacts with the Gemini LLM.
    -   **On-Chain Identity:** Agents have a persistent on-chain identity via a deterministically generated Gnosis Safe wallet, enabling them to interact with protocols like Olas.
-   **Relevant Documentation:** `AGENT_README.md`
