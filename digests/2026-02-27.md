# Contributors Update — Feb 27, 2026

> Convention-over-configuration is the new rule, and we need a release process.

## What changed

- **Fund distribution ported to TS** — worker handles Master Safe -> agent EOA funding natively, replacing the Python middleware for this function. [Commit](https://github.com/Jinn-Network/jinn-cli-agents/commit/88ec717a)
- **Safe dispatch bug** — jobs were being dispatched from the EOA instead of the Safe, causing zero request counts despite hundreds of deliveries. Fixed.
- **Staking dashboard** — owner filter, risk labels, epoch progress, balance views. [PR #192](https://github.com/Jinn-Network/jinn-cli-agents/pull/192)
- **ADW integration** — artifact creation now writes to on-chain ADW registries. Ponder isn't indexing the documents yet — debugging.
- **Orchestrator sandbox** — venture created for testing the UDA orchestrator template. First dispatch burned ~5M tokens, stepping through one-at-a-time now.

## Architecture decisions

- **Gradual TypeScript migration** — whenever we need middleware functionality (funding, staking, service creation), we build it in TS. Single-language stack, easier packaging for operators.
- **Convention over configuration** — non-secret config (venture filter, mech filter, etc.) moving from env vars to a config file. Zero env vars beyond secrets for startup. JINN-404.
- **Tagged releases for jinn-node** — operators pull from main and we can't tell what they're running. Need a release process.
- **Multi-staking-contract support** — explorer needs to support both old and new contracts. JINN-412.

## What's next

- stOLAS: service creation works but needs to also create the mech. Blocked on config update.
- Private IPFS network: implemented, final E2E debugging.
- Content pipeline: commit-highlights -> daily digest -> Telegram, each as a separate venture consuming the previous one's feed.
- Ponder debugging for ADW document indexing.

## Open questions

- When to cut the first tagged release? Monday after staking goes live?
- Config file format — YAML, TOML, JSON? Needs a proposal.
