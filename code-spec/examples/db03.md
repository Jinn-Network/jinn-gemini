# Default Behavior Violation: Using console.* Instead of workerLogger

## ❌ Wrong

```typescript
try {
  await fetch(url);
} catch (e) {
  console.error('Fetch failed:', e);
}
```

## Violations

1. ❌ **Uses `console.error`** instead of `workerLogger.error`
2. ❌ **String concatenation** instead of structured context object
3. ❌ **Missing url in context** - Can't tell which fetch failed
4. ❌ **Error not propagated** - Silently swallows after logging

## Why console.* is Wrong

### Comparison: console vs workerLogger

| `console.error` | `workerLogger.error` |
|-----------------|----------------------|
| Unstructured text | Structured JSON |
| Not queryable | Queryable by fields |
| No telemetry integration | OpenTelemetry compatible |
| No trace correlation | Correlates with spans |
| Inconsistent format | Consistent format |
| Lost in production | Aggregated in logs |

### String Concatenation vs Context Object

```typescript
// ❌ String interpolation - not queryable
console.error(`Failed to fetch ${url}:`, err);

// ✅ Structured object - queryable by field
workerLogger.error('Failed to fetch', { url, error: err });
```

**Why structured is better:**
- Query logs: `logs.where('url', '=', 'https://api.example.com')`
- Aggregate errors by URL
- Correlate with traces via `url` field
- Machine-parseable for alerting

## Real-World Impact

### Scenario: Production Error

**With console.error:**
```
[timestamp] Fetch failed: Error: HTTP 500
```

**What you DON'T know:**
- Which URL failed?
- What was the request context?
- Which user/job was affected?
- Can you query for all failures of this URL?

**With workerLogger.error:**
```json
{
  "timestamp": "2025-01-09T12:34:56Z",
  "level": "error",
  "message": "Fetch failed",
  "url": "https://api.example.com/data",
  "jobId": "req-123",
  "userId": "user-456",
  "error": "HTTP 500: Internal Server Error",
  "stack": "...",
  "traceId": "abc-def-123"
}
```

**Now you CAN:**
- Find all errors for this specific URL
- Correlate with trace in observability tool
- See which job/user was affected
- Build dashboards showing error rates by URL

## Other console.* Violations

```typescript
// ❌ All of these are wrong
console.log('Error happened:', error);
console.warn('Warning:', warning);
console.info('Info:', info);
console.debug('Debug:', debug);

// ✅ Use workerLogger instead
workerLogger.error('Error happened', { error });
workerLogger.warn('Warning occurred', { warning });
workerLogger.info('Info message', { info });
workerLogger.debug('Debug info', { debug });
```

## Correct Version

```typescript
try {
  const response = await fetch(url);
  return response;
} catch (error) {
  workerLogger.error('Fetch failed', {
    url,
    error: error instanceof Error ? error.message : String(error),
    stack: error instanceof Error ? error.stack : undefined
  });
  throw error;
}
```

## When is console.* OK?

**Only during local development for quick debugging:**

```typescript
// Temporary debugging (delete before committing)
console.log('DEBUG: checking value', value);
```

**But even then, workerLogger.debug is better:**

```typescript
// Proper debug logging (safe to commit)
workerLogger.debug('Checking value', { value, step: 'validation' });
```
